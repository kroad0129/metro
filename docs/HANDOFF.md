## 설계에서 반드시 지켜야 할 것

구현하다 보면 되돌리고 싶어지는 결정들이라 이유를 적어둔다.

1. **폴링은 옵트인이다(기본 꺼짐).** "자동 갱신" 스위치로 15초 폴링을 켤 수 있고, 탭이 보일 때만 돈다. 개발키(1000회/일)로 켜둔 채 방치하지 말 것. 기본값을 켜짐으로 바꾸려면 먼저 정식키를 받아야 한다.
2. **`barvlDt`를 카운트다운으로 쓰지 말 것.** (역·방향·거리)별 고정 조회표라 구간을 지나는 동안 값이 안 변하고, `recptnDt`가 갱신돼도 그대로다. 매초 깎으면 폴링마다 시간이 되돌아간다. 남은 시간은 반드시 `virtualTrain`/`segmentTracker` 모델을, 위치는 `placement`(이산 배치)를 거친다. 모델을 건드리면 재생 테스트(`virtualTrain.replay.test.ts`)가 5가지 약속을 검증한다 — 이 테스트를 지우지 말 것.
2-1. **구간 내 위치를 보간하지 말 것.** 몇 % 지점인지는 API가 모르는 정보다. 한 번 연속 보간(virtualGaps)으로 만들었다가 정차·지연·위치미상이 전부 "역에 오래 서 있음"으로 보이는 문제로 걷어냈다(스펙 `-discrete-placement.md`). 역이면 점, 구간이면 화살표 — 이 두 가지만 그린다.
2-2. **화면 구조를 상황에 따라 바꾸지 말 것.** 열차 수·겹침·글자 길이 때문에 줄이 생기거나 높이가 자라거나 자리가 흔들리면 안 된다. 겹치면 아래로 내리는 대신 마크 하나를 두 색으로 그리고, 시간은 고정 두 칸에만 쓴다. 여러 시안을 만들어 비교한 끝에 나온 결론이라(스펙 "반복 4"), 되돌리기 전에 그 기록을 먼저 읽을 것. UI를 만질 때는 `?mock`으로 모든 상황을 조합해 확인한다.
3. **`updnLine`을 쓰지 말 것.** 개화행이 `"하행"`으로 온다. `statnFid`/`statnTid` 비교가 정답이다.
4. **`subwayId` 필터를 빼지 말 것.** 환승역에서 다른 노선 열차가 섞여 온다.
5. **오류 응답의 평평한 구조 처리를 건드리지 말 것.** 이게 깨지면 키 만료·할당량 소진이 조용히 "열차 없음"으로 보인다. 이 앱에서 제일 위험한 실패다.
6. **급행을 백엔드에서 다시 거르지 말 것.** 상류가 이미 걸러준다. 우리가 또 거르면 `isExpressStop`이 틀렸을 때 실제 열차가 사라진다.

### 2026-07-24 리뷰에서 나온 것

**적용됨**
- 급행 통과 표시: 급행이 안 서는 역에 걸치면 "정차"가 아니라 "통과"(`trackMarks.isPassing`). 겹칠 때 딱지는 급행 우선(`groupMarks`의 lead) — 일반 정차 위를 급행이 지나가면 "통과"가 이긴다.
- 죽은 코드 `positionRatio` 제거(연속 보간 시절 잔재, 이산 배치 전환 후 아무도 안 씀). `train-position.ts`도 삭제.
- **[A-2 해결] 정차 중 거짓 "곧 도착" + 추월 오정렬.** 근본 원인은 정렬이 아니라 카운트다운이었다 — 대피로 서 있는 일반의 남은 시간이 바닥(~19초)까지 내려가 거짓 곧도착을 띄우고, 앞서 달리는 급행보다 위로 왔다(`docs/화면 캡처 2026-07-24 095344.png` 증거). 이제 **정차(ARRIVED) 중이면 카운트다운이 줄지 않고 barvlDt에 얼려 있다**(`liveRemainingSeconds`). 출발을 관측하면 `moveStartMs` 기준으로 구간 처음(전체 소요)부터 센다. 남은 시간이 정직해지니 정렬(남은 시간 오름차순)도 자동으로 맞는다 — 위치로 억지 정렬하는 편법을 안 쓴다. **설계 변경**: 열차가 멈추면 남은 시간이 늘 수 있다(실제로 더 걸리므로) — 재생 불변식을 "이동 중엔 안 늘고, 정차하면 늘 수 있다"로 바로잡았다.
- **[지연 제거] `stallSeconds`/`DELAY_NOTICE_SECONDS`/`delayed`/`isDelayExempt` 삭제.** 배지가 화면에서 빠진 뒤로 계산만 되고 아무도 안 썼다. A-2의 "정차하면 남은 시간이 는다"가 지연을 정직하게 대신 드러내므로 별도 배지가 불필요. (되살리려면 git 이력에서.)

**보류(결정·관측 대기)**
- **[B-1] 위상을 상태코드가 아니라 경과시간에서 유도.** 15초 폴링이라 출발/진입 상태를 자주 놓쳐(≈50%) 점이 순간이동한다. `segmentStartedAtMs`+`barvlDt`로 구간 내 위상(출발/운행/진입)을 시간에서 계산하면 매끄러워진다. `arvlCd`는 정차 확정에만.
- **[C-1] 적응형 폴링.** 프론트 폴링 15초(`useTrainData.ts:9`)와 백엔드 캐시 TTL 10초(`configuration.ts:32`)가 어긋나 캐시가 폴링을 못 덮는다 → 폴링마다 실호출(개발키 1000회/일이면 ~4시간 소진). 가까운 열차 있을 때만 촘촘히 + 캐시 TTL을 폴링과 정렬하면 호출을 줄이면서 중요한 순간엔 더 자주. **지금은 기본값 고정, 이 항목은 기록만.**

### 나중에 (우선순위 논의됨)

정식키 → 폴링 기본 켜짐 → `realtimePosition`으로 열차 깜빡임 보강(증상 생기면) →
**걷는 시간 → 행동 판단**("지금 나가", "뛰어" — 차별점 핵심) → 다른 노선 추가 → PWA → 홈 화면 위젯.

## 서울시 API에 대해 알아낸 것
