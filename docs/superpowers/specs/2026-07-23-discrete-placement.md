# 이산 배치 모델 — 위치 보간을 버리고 상태만 그린다

날짜: 2026-07-23 (저녁) · 사용자 제안으로 방향 전환

## 왜 바꿨나

사용자 관찰: "역에 오래 정차해 있는 것 같다, 뭔가 의심스럽다."
실녹화(replay-jeungmi.json) 분석으로 확인한 사실:

- 실제 정차는 20~40초인데 API "도착" 상태는 30~60초+ 지속된다(갱신 지연 10~27초 포함).
- 더 크게는, 연속 보간 모델(virtualGaps)이 "위치를 모르면 점을 세워두는" 설계라
  **정차·지연·위치미상이 화면에서 전부 "역에 서 있음"으로 보였다.**
  예: 9129는 당산 부근 "운행중" 상태로 180초 — 이 안에 정차와 주행이 섞여 있다.

구간 내 몇 % 지점인지는 API가 주지 않는 정보다. 흉내내는 비용(이동 진행률, 최소
주행시간 보정, 순간이동 방지…)이 전부 이 거짓말을 유지하는 비용이었다.

## 무엇으로 바꿨나

시각 언어 3개 (`placement.ts` + `TrainFlow.tsx`):

| 기호 | 상태(arvlCd) | 의미 |
|---|---|---|
| ● 점 + "도착" | 도착·전역도착 | 그 역에 서 있음 |
| ◎ 맥동 점 (+ 내 역이면 "곧 도착") | 진입·전역진입 | 역에 들어서는 중 |
| `› › ›` 흐르는 화살표 | 출발·전역출발·운행중 | 두 역 사이 구간 이동 중 (위치 주장 없음) |

- `trainPlacement(train)` → `{station, gap, arriving}` \| `{segment, fromGap, toGap}` \| null(지나감)
- 화살표는 구간 폭 전체에 걸치고 밝기 파도가 진행 방향(오른쪽)으로 흐른다.
- 남은 시간(가상 열차 시간 모델)·지연 배지·급행 색은 그대로 유지.
- `virtualGaps`(연속 보간)는 삭제. 시간 모델(`liveRemainingSeconds`)과
  `stallSeconds`는 유지 — 지연은 이제 "오래 흐르는 화살표 + 지연 배지"로 보인다.

## 알려진 한계 (데이터의 한계)

- 전전역 이상(d≥2)은 "운행중"만 와서 정차·이동을 구분 못 한다 — 구간 이동으로
  그리는 게 평균적으로 정직하다(정차 ~30초 vs 구간 90~130초).
  → 상류 역 추가 조회(정식키 이후 계획)로 정확해진다. placement는 상태만 보므로
  그때도 코드 변경 없이 맞아진다.

## 반복 2 — 사용자 피드백 반영 (같은 날)

피드백: "가는 중 애니메이션은 괜찮은데, 출발하거나 도착할 때쯤은 이게 뭔가 싶다.
그리고 라인이 겹쳤을 때 대비가 하나도 안 되어 있다."

1. **상태를 글자로도 말한다** — 기호(맥동·전환)만으로는 안 읽힌다:
   - 중간 역: 서 있으면 **"정차"**, 들어서는 중이면 **"진입"** (점 위에, 열차 색).
     "도착"이라 쓰면 내 역 도착과 헷갈려서 단어를 분리했다.
   - 내 역: "도착" / "곧 도착" 그대로 (시간 자리 대체).
   - 막 출발(DEPARTED): 화살표 위에 **"출발"** — 점이 화살표로 바뀐 이유를 설명한다.
     운행중(TRAVELING)은 글자 없음(화살표로 충분, 사용자 확인).
2. **겹침 대비: 레인(lane) 시스템** (`assignLanes`) — 시각적 자리(점: 역 ±8%,
   화살표: 구간 가운데 ±10%)가 겹치는 열차는 한 줄(2.8rem)씩 아래로 내려 그린다.
   급행이 일반과 같은 구간을 달리는 경우가 흔한 실사례. 트랙 높이는 줄 수만큼 늘어난다.
   인접 구간(끝점만 닿음)은 겹침으로 치지 않는다.

## 반복 3 — 구간 3분할 (사용자 제안)

"출발이나 진입은 역과 역을 3분할한다고 생각하고 표시하면 어때?" — 채택.
상태의 의미가 이미 구간의 위상을 말하고 있어서, 지어내는 위치가 아니다:

- **출발(DEPARTED)** = 구간 첫 ⅓ (떠난 역 직후) + "출발"
- **운행중(TRAVELING)** = 가운데 ⅓
- **진입(APPROACHING)** = 다음 역으로 가는 구간의 마지막 ⅓ + "진입"(내 역이면 "곧 도착")

이로써 진입은 역 위의 맥동 점이 아니라 구간 끝자락의 화살표가 됐고(맥동 제거),
점(TrainMarker)은 정차(ARRIVED) 전용이 됐다. 상태가 바뀔 때마다 화살표가 ⅓씩
전진해 보간 없이도 진행감이 생긴다. 같은 구간의 출발·진입 열차가 자연히 다른
자리에 그려져 겹침도 줄었다. 재생 검증의 순서 축은 위상 중앙(from−⅙/−½/−⅚)으로
정의했고, 실녹화에서 역행 없음을 확인했다.

## 재생 검증(replay)의 새 약속

1. 남은 시간은 절대 늘어나지 않는다.
2. 배치는 절대 뒤로 가지 않는다 (placementOrder 축).
3. 폴링 순간의 남은 시간 하향 점프는 작다 (첫 전이만 150초, 이후 40초).
4. 역에 서 있는 열차(정차·진입)는 지연으로 표시되지 않는다.
5. 실제 지연(9129 당산→선유도, 추정 125초/실측 191초)은 감지된다.

재생 검증이 잡은 결함: stallSeconds가 "내 역 진입(d=0)"을 옛 모델 기준(이동)으로
봐서 진입 중인 열차에 지연이 붙었다 → isParked를 배치 모델과 같은 기준
(도착·진입 = 정지)으로 통일.
