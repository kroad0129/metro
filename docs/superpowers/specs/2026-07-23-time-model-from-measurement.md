# 시간·위치 모델 재설계 — 실측 근거

날짜: 2026-07-23
상태: 구현 완료

## 측정

증미·가양·여의도 3개 역을 10초 간격으로 ~3.7분 관측(`scratchpad/cadence.jsonl`). 핵심 발견:

**`barvlDt`(선택역까지 남은 초)는 실시간 카운트다운이 아니라 "위상별 고정 추정치"다.**

- 전역(1정거장 전)에 있는 열차: **항상 95~100초** (측정 32건 전부, 평균 95).
- 전전역: ~220초. 내역 진입: ~20초. 위상마다 고정값이고, 열차가 다음 위상으로 넘어갈 때만 툭 바뀐다.
- 열차 9075는 등촌(전역)에서 **132초 동안 지연**됐는데 barvlDt는 내내 95로 고정 → 다음 위상(진입)에서 20으로 하강.
- barvlDt가 **위로 튄(지연을 값 증가로 표현한) 사례: 0건.** 지연은 "전역에 오래 얼어있는" 형태로만 나타난다.

## 이전 모델의 문제

barvlDt를 매초 −1로 깎아 부드러운 카운트다운/점 이동을 만들었는데:
- 서버 값이 얼어 있어 폴링마다 되돌아가거나(줄었다 늘었다), 심하면 로컬 카운트다운이 0까지 내려가 **열차가 전역에 서 있는데 "도착"으로 표시**되는 오류.
- 급행 지연도 barvlDt가 안 변하니 반영 불가.

## 새 모델 (구현됨)

barvlDt로 연속 카운트다운을 만드는 건 원천 불가 → **가짜 카운트다운을 버린다.**

1. **점 위치 = 실제 있는 역(arvlMsg3) + 위상 미세보정.** `trainLeftPercent(track, train)`이 현재역 index에 status 보정(진입 −0.35 / 정차 0 / 출발 +0.35 / 운행 0)을 더한다. 시간 미사용. 지연되면 점이 그 역에 그대로 머문다(거짓 도착 없음). 역을 넘을 때만(폴링 시) 이동하고 CSS transition이 부드럽게 잇는다.
2. **시간 = barvlDt를 분 단위 올림(ceil).** `formatRemaining`: 95초 → "2분"(사용자 기대와 일치). 초는 얼어있어 무의미하므로 버린다. 폴링(15초)마다 갱신.
3. **급행 지연은 자동 반영.** 지연된 일반 열차는 "전역에 오래 머무는 점 + 도착"으로 정확히 보인다 — 별도 모델링 없이.

## 제거된 것
- `SECONDS_PER_GAP`, 시간 기반 `trainLeftPercent`, `remainingAt`(브라우저 카운트다운), 단조 보정(`monotonicRemaining` 전체), `ARRIVAL_SOON_SECONDS`. `useNow`는 "N초 전 갱신" 표시용으로만 남는다.

## 트레이드오프
초 단위 실시간 카운트다운(초기 요청 기능)을 포기했다. 실측이 그게 허구임을 보여줬고, 사용자 우선순위가 "정확·불편 없음"으로 바뀌었기 때문. 대신 폴링을 켜면 15초마다 갱신되어 충분히 살아있다.

## 가상 열차 모델 (2026-07-23 확정, CSS-cap 보간을 대체)

CSS-cap 보간은 트랜지션 시간이 이동거리와 무관해 "홱 튀는" 문제가 있었고, 시간 표시가 폴링
사이 얼어 있었다. 이를 **클라이언트 시뮬레이션(dead reckoning + 관측 보정)**으로 대체했다 —
`src/utils/virtualTrain.ts`.

원리 (실제 지하철 운행 특성을 그대로 모델링):
- **페이스는 운영사 추정치에서 유도**: barvlDt 위상 간격 220→95→20 ⇒ 한 정거장 ≈ 110초
  (`SECONDS_PER_GAP`), 출발→진입 75초, 진입→도착 15초.
- **앵커**: `useTrainData`가 폴링마다 열차별 위상(현재역+상태)의 시작 시각(`anchorSinceMs`)을
  기록. 위상이 같으면 유지, 바뀌면 갱신. 역 변경 시 초기화.
- **전진**: `virtualGaps(status, d, anchorSinceMs, nowMs)`가 위상 시작점부터 페이스대로 매초
  전진(useNow 틱). 위치 단위는 "선택역까지 남은 정거장 수(gaps)".
- **상한(cap)**: 다음 위상 확인 전엔 구간 끝 직전(0.15 gap 앞)에서 멈춰 대기 — 전광판이 폐색
  확인 전 구간 안에서만 움직이는 것과 동일. 거짓 도착 불가, 지연은 "역 앞 대기"로 보인다.
- **정차(ARRIVED)는 자동 출발 금지** — 출발 위상 확인이 와야 움직인다(급행 대기 지연이 정확히 보임).
- **시간 = gaps × 110초**: 점과 시간이 한 모델에서 나와 모순 불가, **매초 초 단위로 줄어든다**
  ("3분 얼어있음" 해결). 표기는 "1분 50초"처럼 분+초(`formatRemaining`).
- 지나간 열차(gaps < 0)는 그리지 않는다(내 역 출발 열차 유령 문제도 해결).
- off-track "다음 열차"는 barvlDt 그대로(원거리 추정은 운영사 값이 낫고, 트랙 진입 경계에서
  모델값과 일치: 2정거장 = 220초).

향후: 열차가 희소하거나 깜빡이면 `realtimePosition`(열차위치정보)으로 연속성 보강.
그 다음 킬러 기능은 "걷는 시간 → 행동 판단"([[upstream-station-query-for-stop-status]]와 별개).
